Building Your First LLM-Powered Chatbot: A Step-by-Step Guide for 2025

By Alex Chen | Published on September 5, 2025 | 12 min read

The rise of powerful, accessible Large Language Models (LLMs) has been a game-changer for developers. What once required a team of machine learning PhDs can now be accomplished with a few lines of Python and an API key. In this guide, we'll walk you through building a simple but functional command-line chatbot that remembers your conversation.

By the end of this tutorial, you will have a working Python script that can hold a multi-turn conversation, forming the perfect foundation for more complex projects. Let's get started!

Step 1: Prerequisites & Environment Setup

Before we write any code, let's make sure you have everything you need. This guide assumes you have Python 3.8 or newer installed.

  1. Get an API Key: We'll use the OpenAI API for this example. You'll need to create an account on the OpenAI Platform and generate a new secret key from your API keys page.
  2. Set Up a Project Folder: Create a new folder for your project (e.g., `llm_chatbot`) and navigate into it in your terminal.
  3. Create a Virtual Environment: It's best practice to use a virtual environment to manage project dependencies. 
    # For macOS/Linux
python3 -m venv venv
source venv/bin/activate

# For Windows
python -m venv venv
.\venv\Scripts\activate
  4. Install the Library: We just need one package to start. 
    pip install openai

Security Note: Never hardcode your API key directly in your script. The safest way is to set it as an environment variable.

Step 2: The Basic Stateless Chatbot

Our first version will be simple: it takes a single user input, sends it to the API, and prints the response. It won't have any memory of past interactions.

Create a new file called `chatbot_v1.py` and add the following code:

import os
from openai import OpenAI

# It's best practice to use environment variables for your API key
# You can set it in your terminal like this: export OPENAI_API_KEY='your-key-here'
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

def get_response(user_prompt):
    try:
        response = client.chat.completions.create(
            model="gpt-3.5-turbo",
            messages=[
                {"role": "system", "content": "You are a helpful assistant."},
                {"role": "user", "content": user_prompt}
            ]
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"An error occurred: {e}"

# Main chat loop
print("Chatbot initialized. Type 'quit' to exit.")
while True:
    user_input = input("You: ")
    if user_input.lower() == 'quit':
        break

    bot_response = get_response(user_input)
    print(f"Bot: {bot_response}")

You can run this script from your terminal with `python chatbot_v1.py`. Try asking it a question! Then ask a follow-up. You'll notice it has no idea what you talked about beforeâ€”let's fix that.

Step 3: Adding Memory for a Real Conversation

To make our chatbot truly conversational, it needs to remember the context of the dialogue. The API allows this by sending the entire conversation history with each new request.

We'll modify our script to store the chat history in a list. Create a new file `chatbot_v2.py`.

import os
from openai import OpenAI

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# Initialize a list to store the conversation history
conversation_history = [
    {"role": "system", "content": "You are a helpful and witty assistant."}
]

def get_contextual_response(user_prompt):
    # Add the user's new message to the history
    conversation_history.append({"role": "user", "content": user_prompt})

    try:
        response = client.chat.completions.create(
            model="gpt-3.5-turbo",
            messages=conversation_history # Send the entire history
        )

        bot_message = response.choices[0].message.content
        # Add the bot's response to the history
        conversation_history.append({"role": "assistant", "content": bot_message})

        return bot_message
    except Exception as e:
        return f"An error occurred: {e}"

# Main chat loop
print("Contextual Chatbot initialized. Type 'quit' to exit.")
while True:
    user_input = input("You: ")
    if user_input.lower() == 'quit':
        break

    bot_response = get_contextual_response(user_input)
    print(f"Bot: {bot_response}")

Now, run this new version with `python chatbot_v2.py`. Ask it "What is the capital of France?". After it answers, ask "And what is its population?". This time, it will know you're referring to Paris. You've successfully given your chatbot memory!

Step 4: Next Steps and Improvements

This simple script is a powerful starting point. Here are some ideas for where to go next:

  • Web Interface: Use a web framework like Flask or FastAPI to build a web-based UI for your chatbot.
  • Connect to Custom Data: Explore techniques like Retrieval-Augmented Generation (RAG) to allow your chatbot to answer questions based on your own documents.
  • Error Handling: Add more robust error handling for API timeouts, rate limits, or invalid keys.
  • Streaming Responses: For longer answers, implement streaming to display the response word-by-word, just like ChatGPT does.
